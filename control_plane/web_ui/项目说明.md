# AI政务业务系统 - 后台管理系统 (admin-web)

> 基于 vue-vben-admin 5.5.9 (Element Plus 版本)

## 📋 项目说明

本项目是**AI政务业务系统**的后台管理端，采用 **vue-vben-admin** 作为基础框架。

### 🏗️ 项目结构说明

admin-web 采用 **monorepo** 结构（这是合理的架构选择）：

```
admin-web/                          # 后台管理系统根目录
├── apps/                           # 应用目录
│   ├── web-ele/                    # Element Plus 版本 ⭐ 主要开发目录
│   │   ├── src/
│   │   │   ├── views/              # ⭐ 在这里添加业务页面
│   │   │   ├── router/             # ⭐ 在这里配置路由
│   │   │   ├── api/                # ⭐ 在这里添加 API
│   │   │   ├── locales/            # 国际化
│   │   │   └── ...
│   │   └── package.json
│   ├── web-antd/                   # Ant Design 版本（可选）
│   └── web-naive/                  # Naive UI 版本（可选）
│
├── packages/                       # 共享包（框架功能）
│   ├── @core/                      # 核心组件
│   ├── effects/                    # 布局、表单、表格等
│   ├── locales/                    # 国际化
│   └── ...
│
├── internal/                       # 内部工具
├── package.json                    # 根配置
├── pnpm-workspace.yaml             # workspace 配置
└── 业务开发指南.md                  # ⭐ 开发文档
```

## 🚀 快速开始

### 1. 安装依赖

```bash
cd admin-web
pnpm install
```

### 2. 启动开发服务器

```bash
# 启动开发服务器（默认 Element Plus 版本）
pnpm dev

# 或指定具体版本
pnpm dev:ele          # Element Plus 版本（推荐）
pnpm dev:antd         # Ant Design 版本
pnpm dev:naive        # Naive UI 版本
pnpm dev:play         # Playground（功能演示）
```

### 3. 访问系统

服务启动后会自动打开浏览器，默认地址：
- **http://localhost:5666** (或其他可用端口)

### 4. 登录

- **用户名**: `vben`
- **密码**: `123456`

## 🏗️ 为什么使用 Monorepo？

虽然其他前端项目（business-mobile、consumer-mobile 等）使用简单结构，但 **admin-web** 使用 monorepo 是合理的：

### 优点

✅ **功能完整**: 后台管理需要大量复杂功能（权限、工作流、表单、表格等）

✅ **代码复用**: 多个版本（Element Plus、Ant Design、Naive UI）共享核心代码

✅ **易于维护**: 框架功能和业务代码分离清晰

✅ **持续更新**: 可以方便地更新 vue-vben-admin 框架

### 开发体验

对于开发人员来说：
- 实际开发只在 `apps/web-ele/src/` 目录
- 不需要关心 packages 和 internal
- 就像开发普通项目一样

## 🆘 常见问题

### Q1: 为什么 admin-web 结构比其他前端项目复杂？

**A**: 因为后台管理需要的功能最复杂（权限、工作流、表单、表格、图表等），monorepo 结构更适合管理这些复杂功能。

### Q2: 开发时需要关心 packages 和 internal 吗？

**A**: 不需要！实际开发只在 `apps/web-ele/src/` 目录，其他都是框架代码。

### Q3: 如何添加业务页面？

**A**: 查看 [业务开发指南.md](./业务开发指南.md)，有完整的步骤说明。

### Q4: 可以改用其他 UI 框架吗？

**A**: 可以！项目同时支持 Element Plus、Ant Design 和 Naive UI。
